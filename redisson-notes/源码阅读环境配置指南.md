# Redisson 源码阅读环境配置指南

## 📋 概述

本文档详细说明了如何配置 Redisson 源码阅读环境，使测试程序能够直接使用本地编译的源码，而不是 Maven 仓库中的 jar 包。

## 🎯 核心目标

- **使用本地源码**：测试程序直接依赖本地编译的 Redisson 源码
- **支持调试**：可以在 IDE 中设置断点、单步调试源码
- **版本兼容**：确保 Java 8 兼容性
- **依赖管理**：正确处理 Maven 依赖解析

## ⚠️ 关键问题与解决方案

### 问题1：版本不匹配导致依赖错误

**问题描述**：
```xml
<!-- 父 POM -->
<version>3.17.6</version>

<!-- redisson 模块 -->
<parent>
    <version>3.50.0</version>  <!-- ❌ 版本不匹配！ -->
</parent>

<!-- redisson-notes 依赖 -->
<dependency>
    <version>${project.version}</version>  <!-- 解析为 3.17.6 -->
</dependency>
```

**解决方案**：
```xml
<!-- 统一所有模块版本 -->
<version>3.17.6</version>
```

### 问题2：Maven 依赖解析优先级

**Maven 依赖解析顺序**：
1. **本地仓库** (`~/.m2/repository/`) - 最高优先级
2. **远程仓库** (Maven Central, 阿里云等)
3. **项目模块** (多模块项目中的其他模块)

**关键点**：
- 当版本不匹配时，Maven 会去远程仓库下载，而不是使用本地编译的源码
- 必须确保编译版本与依赖版本完全一致

### 问题3：Java 版本兼容性

**问题**：
- Redisson 3.50.0 需要 Java 17+
- 我们需要 Java 8 兼容性

**解决方案**：
- 使用 Redisson 3.17.6（最后一个支持 Java 8 的版本）
- 降级相关依赖版本

## 🔧 配置步骤

### 步骤1：统一版本号

```xml
<!-- 父 POM (pom.xml) -->
<version>3.17.6</version>

<!-- redisson 模块 (redisson/pom.xml) -->
<parent>
    <version>3.17.6</version>
</parent>
```

### 步骤2：解决 Java 8 兼容性

```xml
<!-- redisson/pom.xml -->
<properties>
    <!-- 降级 Spring 版本以支持 Java 8 -->
    <spring.version>5.3.23</spring.version>
    <spring-session.version>2.7.3</spring-session.version>
    <spring-boot-actuator.version>2.7.18</spring-boot-actuator.version>
    <spring-boot-autoconfigure.version>2.7.18</spring-boot-autoconfigure.version>
    <caffeine.version>2.9.3</caffeine.version>
</properties>
```

### 步骤3：编译本地源码

```bash
# 设置 Java 8 环境
export JAVA_HOME=/Library/Java/JavaVirtualMachines/zulu-8.jdk/Contents/Home

# 编译 redisson 模块
mvn package -pl redisson -DskipTests -Dmaven.javadoc.skip=true -Dcheckstyle.skip=true -Dmaven.test.skip=true -Dpmd.skip=true
```

### 步骤4：安装到本地仓库

```bash
# 安装编译的 jar 到本地 Maven 仓库
mvn install:install-file -Dfile=redisson/target/redisson-3.17.6.jar -DgroupId=org.redisson -DartifactId=redisson -Dversion=3.17.6 -Dpackaging=jar
```

### 步骤5：配置测试依赖

```xml
<!-- redisson-notes/pom.xml -->
<dependencies>
    <!-- 使用本地源码编译的 redisson -->
    <dependency>
        <groupId>org.redisson</groupId>
        <artifactId>redisson</artifactId>
        <version>${project.version}</version>  <!-- 解析为 3.17.6 -->
    </dependency>
    
    <!-- 必要的依赖 -->
    <dependency>
        <groupId>com.fasterxml.jackson.datatype</groupId>
        <artifactId>jackson-datatype-jsr310</artifactId>
        <version>2.13.5</version>
    </dependency>
    
    <dependency>
        <groupId>com.esotericsoftware</groupId>
        <artifactId>kryo</artifactId>
        <version>3.0.3</version>
    </dependency>
</dependencies>
```

## 🚀 验证配置

### 创建版本测试

```java
@Test
public void testRedissonVersion() {
    RedissonClient client = RedisTestHelper.createSingleServerClient();
    
    // 测试基本操作
    client.getBucket("version_test").set("Hello from local source!");
    String value = (String) client.getBucket("version_test").get();
    assertEquals("Hello from local source!", value);
    
    System.out.println("✅ 成功使用本地编译的 Redisson 3.17.6 源码！");
    System.out.println("✅ 测试程序现在直接依赖本地源码编译的类");
}
```

### 运行测试

```bash
mvn test -pl redisson-notes -Dtest="VersionTest#testRedissonVersion"
```

## 📝 使用指南

### 修改源码

1. 直接编辑 `redisson/src/main/java/` 下的文件
2. 重新编译：
   ```bash
   mvn package -pl redisson -DskipTests -Dmaven.javadoc.skip=true -Dcheckstyle.skip=true -Dmaven.test.skip=true -Dpmd.skip=true
   ```
3. 运行测试：
   ```bash
   mvn test -pl redisson-notes
   ```

### 调试源码

- 在 IDE 中设置断点
- 单步调试 Redisson 源码
- 查看变量值和调用栈

## ⚠️ 注意事项

### 1. 版本一致性
- 所有模块的版本必须完全一致
- 编译版本与依赖版本必须匹配

### 2. 依赖管理
- 本地编译的 jar 会覆盖远程仓库的版本
- 确保本地仓库中有正确的版本

### 3. Java 版本
- 使用 Java 8 编译和运行
- 确保环境变量设置正确

### 4. 依赖冲突
- 注意 Spring 版本兼容性
- 避免依赖版本冲突

## 🔍 故障排除

### 问题：仍然使用远程 jar 包
**原因**：版本不匹配
**解决**：检查所有模块版本是否一致

### 问题：编译失败
**原因**：Java 版本不兼容
**解决**：使用 Java 8 并降级相关依赖

### 问题：运行时 ClassNotFoundException
**原因**：缺少必要依赖
**解决**：添加 Jackson JSR310、Kryo 等依赖
